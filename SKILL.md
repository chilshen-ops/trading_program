## ⚠️ open-long / open-short 命令格式（关键！）

```bash
# ✅ 正确格式：保证金是位置参数，放在命令名后面
python3 trader.py open-long 10 --symbol=XXX --leverage=10
                            ^ 位置参数，不带 --


# ❌ 错误格式：--margin 会报 "unrecognized arguments"
python3 trader.py open-long --margin 10 --symbol=XXX
```


| 参数 | 必须 | 类型 | 说明 |
|------|------|------|------|
| `margin` | ✅ | 位置参数 | **保证金金额（USDT）**，不带 `--` |
| `--symbol` | 是 | 选项 | 币种，如 `NILUSDT` |
| `--leverage` | 否 | 选项 | 杠杆倍数，默认 `10` |

## 🛡️ 自动止损止盈

> ⚠️ **开仓成功后自动设置 2% 止损（不含杠杆）作为兜底保护，LLM 判断盈利后接管，重新设置止盈/新止损**

> ⚠️ **PM 账户限制**：API 无法查询/取消条件单，取消失败不影响设置新单（会自动覆盖）

| 操作 | 止损 | 后续止盈 |
|------|------|---------|
| **做多（open-long）** | 跌破入场价 2% 时触发，市价平仓 | LLM 分析盈利后重新设置 |
| **做空（open-short）** | 涨过入场价 2% 时触发，市价平仓 | LLM 分析盈利后重新设置 |

### 流程说明

**1. 开仓即设2% 止损**
- 这是地板保护，防止开盘就爆
- 止损触发后市价平仓

**2. LLM 分析后接管**
- 当持仓出现盈利，LLM 调用 `llm-hold` 分析盘口
- 根据 K线形态、成交量、流动性密集区重新判断止损位
- 大模型只需给程序一个参数+价格，程序自动取消旧单并下新单

**3. replace-order 命令（自动取消旧单后下新单）**

> ⚠️ PM 账户限制：取消可能返回 "Unknown order"（Binance 端未同步），但不影响设置新单
### LLM 接管后的止盈/止损操作（直接设置）

> ⚠️ 一键命令：自动取消旧单 + 设置新单



```bash
# 设置止损（基于入场价，自动获取持仓）
# --price 支持绝对价格或百分比
python3 trader.py replace-order --symbol=XXX --side=SELL --price=1% --type=STOP_MARKET    # 平多：入场价下方1%（-1%浮亏止损）
python3 trader.py replace-order --symbol=XXX --side=BUY  --price=1% --type=STOP_MARKET    # 平空：入场价上方1%（+1%浮亏止损）
python3 trader.py replace-order --symbol=XXX --side=SELL --price=0.070 --type=STOP_MARKET  # 绝对价格（直接指定触发价）
```

**`--price` 参数支持两种格式（基于入场价，非当前价）**：

| 格式 | 示例 | 说明 |
|------|------|------|
| **百分比** | `1%` / `2%` | 基于入场价：`SELL`=入场价×(1-pct/100)，`BUY`=入场价×(1+pct/100) |
| **绝对价格** | `0.070` / `50000` | 直接用作触发价格 |

**百分比自动计算规则**：

| --side | 效果 |
|--------|------|
| `SELL`（平多仓） | 触发价 = 入场价 × (1 - pct/100)，即 **浮亏 pct%** 触发止损 |
| `BUY`（平空仓） | 触发价 = 入场价 × (1 + pct/100)，即 **浮亏 pct%** 触发止损 |

**追踪机制：**
- 活跃 algo_id 保存到 `.conditional_orders.json`（按 symbol + side 索引）
- 每次 replace 自动查找旧单 → 尝试取消 → 下新单 → 更新追踪文件
- 取消失败（可能已触发）不影响下新单

**4. cancel-conditionals 命令（按币种取消追踪的委托）**

```bash
# 手动平仓时同时取消全部追踪委托（SELL + BUY）
python3 trader.py cancel-conditionals --symbol=NILUSDT

# 只取消 NILUSDT 的 SELL 委托（平多），保留 BUY
python3 trader.py cancel-conditionals --symbol=NILUSDT --side=SELL
```

**使用场景：**
- 需要手动干预时，先取消程序追踪的旧单
- 换方向交易前，先清掉旧的止损
- 程序重启后，追踪文件仍保留历史 algo_id（重复取消无害）

### 步骤4：执行

#### 4.1 开仓

**执行前确认**：
- 步骤3输出方向=做多或做空 → 执行
- 步骤3输出方向=不做 → 跳过
- 读取 `blacklist.json`（禁止交易的下架币种）

**仓位和杠杆**：
- 完全由 LLM 在步骤3的分析中自行决定仓位比例（0-100%）
- 程序按 LLM 指定的仓位和杠杆执行，**换算公式**：
  ```
  开仓保证金(USDT) = 总资产 × 仓位% / 杠杆
  例：总资产1000U, 仓位50%, 杠杆10x → 开仓保证金 = 1000 × 50% / 10 = 50 USDT
  ```
- **0仓位规则**：LLM 说50%就开50%，说80%就开80%。没有任何强制仓位公式，但 LLM 必须自己给出明确的仓位百分比和杠杆倍数。

**命令**：
```bash
python3 trader.py open-long 10 --symbol=XXX --leverage=10      # 开多
python3 trader.py open-short 10 --symbol=XXX --leverage=10      # 开空
```
- 确认开仓成功，否则重试
- 开仓后自动设置2%地板止损

#### 4.2 持仓管理

执行以下检查
> 持仓期间每次巡检按顺序检查：

**检查1：开仓理由还在吗？（按lg.md的持仓管理逻辑）**
- 回顾开仓时的分析逻辑，新数据还支持吗？
- 30m方向是否改变？5m方向是否延续？
- 连续2根K线不推进 → 立即退出，不等止损

**检查2：硬止损** — 逻辑与事实背离 → 立即全平

**检查3：阶梯止损（纯百分比设计）**

> 所有止损触发价均以**不含杠杆的浮盈百分比**为激活条件，止损位置以**入场价的百分比**表达，`replace-order --price=X%` 自动换算为绝对价格。

**做多（LONG）阶梯止损：**

| 台阶 | 触发条件（浮盈不含杠杆） | 止损位置（入场价 ×） | replace-order 命令 | 说明 |
|------|------------------------|-------------------|-------------------|------|
| T0 初始 | 开仓即设 | 入场价 × 0.98（-2%） | `--price=2%` | 地板保护，防止开盘爆 |
| T1 保本 | 浮盈 >= +1% | 入场价 × 1.00（±0%） | `--price=0%` | 涨到入场价上方，止损移到保本 |
| T2 锁利 | 浮盈 >= +2% | 入场价 × 1.01（+1%） | `--price=1%` | 锁定1%利润 |
| T3 追踪 | 浮盈 >= +3% | 入场价 × 1.02（+2%） | `--price=2%` | 追踪止损，锁定2% |
| T4 追踪 | 浮盈 >= +4% | 入场价 × 1.03（+3%） | `--price=3%` | 继续上移 |
| T5 追踪 | 浮盈 >= +5% | 入场价 × 1.04（+4%） | `--price=4%` | 继续上移 |

> 每涨1%浮盈，止损上移1%，循环至市价触发。

**做空（SHORT）阶梯止损：**

| 台阶 | 触发条件（浮盈不含杠杆） | 止损位置（入场价 ×） | replace-order 命令 | 说明 |
|------|------------------------|-------------------|-------------------|------|
| T0 初始 | 开仓即设 | 入场价 × 1.02（+2%） | `--price=2%` | 地板保护 |
| T1 保本 | 浮盈 >= +1% | 入场价 × 1.00（±0%） | `--price=0%` | 跌到入场价下方，止损移到保本 |
| T2 锁利 | 浮盈 >= +2% | 入场价 × 0.99（-1%） | `--price=1%` | 锁定1%利润 |
| T3 追踪 | 浮盈 >= +3% | 入场价 × 0.98（-2%） | `--price=2%` | 追踪止损 |
| T4 追踪 | 浮盈 >= +4% | 入场价 × 0.97（-3%） | `--price=3%` | 继续下移 |
| T5 追踪 | 浮盈 >= +5% | 入场价 × 0.96（-4%） | `--price=4%` | 继续下移 |

> 每跌1%浮盈，止损下移1%，循环至市价触发。

**`replace-order` 百分比 → 绝对价格换算规则：**

| --side | `--price=X%` | 实际效果 |
|--------|-------------|---------|
| `SELL`（平多） | `X%` | 触发价 = **入场价** × (1 - X/100) |
| `BUY`（平空） | `X%` | 触发价 = **入场价** × (1 + X/100) |

**执行示例：**

```bash
# 做多，开仓价 $100，当前浮盈 +2%
replace-order --symbol=XXX --side=SELL --price=1%   # 止损: 100×0.99=$99，锁定1%利润

# 做空，开仓价 $100，当前浮盈 +3%
replace-order --symbol=XXX --side=BUY --price=2%   # 止损: 100×1.02=$102，锁定2%利润
```

> ⚠️ **入场价获取**：程序自动从持仓数据读取 `entryPrice`，无需手动指定。

```bash
# 设置止损（基于入场价，自动获取持仓）
python3 trader.py replace-order --symbol=XXX --side=SELL --price=1% --type=STOP_MARKET    # 平多：入场价下方1%（-1%浮亏止损）
python3 trader.py replace-order --symbol=XXX --side=BUY  --price=1% --type=STOP_MARKET    # 平空：入场价上方1%（+1%浮亏止损）
```

### 步骤6：检查表和汇报（持仓时必须汇报，空仓跳过）

#### 6.1 盈亏计算
- 多头：盈亏率% = (当前价 - 入场价) / 入场价 × 杠杆倍数 × 100
- 空头：盈亏率% = (入场价 - 当前价) / 入场价 × 杠杆倍数 × 100

#### 6.2 持仓状态
- 持仓币种、方向、仓位比例、杠杆倍数
- 入场价格、当前价格
- **盈亏率%（必须填写）**、绝对盈亏$

#### 6.3 检查清单执行结果（逐项列出）

| 检查项 | 结论（不含杠杆） | 备注（不含杠杆） |
|--------|------|------|
| 开仓理由检验 | 还在延续 / 2根没推进 / 反转 | 检验结果 |
| 硬止损检查 | 通过 / 触发 | - |
| 保本止损检查 | 通过 / 触发（已移） | 浮盈 vs +1%激活线 |
| 阶梯止损状态 | 未激活 / 已激活(X.X%) / 触发 | 当前阶梯 vs 浮盈 |
| 阶梯止损触发 | 通过 / 触发 | 当前价 vs TP_trigger |
| 方向延续性检查 | 信号是否持续 | 差在哪里 |
| 本轮动作 | 无动作 / 已平仓 | - |

#### 6.4 距止损/止盈位距离
- 距硬止损：XX%（当前价 vs 止损价）
- 距保本止损：XX%（当前止损 vs 入场价）
- 距阶梯触发线：XX%（当前价 vs TP_trigger）
- 当前阶梯：XX%

#### 6.5 交易结果（如有平仓）
- 平仓价格、比例
- 盈亏金额、盈亏率%
- 平仓原因（硬止损/保本止损/阶梯止损/动量衰竭/2根未推进/反向爆发）

## 🏦 统一账户（PM_2）交易用法

- `trader.py` 自动检测账户类型并路由，无需手动切换
- 统一账户使用 `papi/v1/um/order` 接口（单向持仓）
- PM账户支持止损止盈条件单（papi/v1/um/algo/order）
- ⚠️ PM账户**无法通过API查询**活跃条件单（接口返回404），`get_conditional_orders()` 不可用
- `replace-order` 和 `cancel-conditionals` 通过 `.conditional_orders.json` 文件追踪 algo_id，自动处理旧单取消
