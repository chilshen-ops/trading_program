```bash
cd ~/.openclaw/workspace/trading_program

# =============================================
# 扫描候选币种（多线程并发，推荐配置）
# =============================================

# =============================================

# =============================================
# LLM 分析
# =============================================
python3 trader.py llm-hold --symbol=XXX      # LLM分析持仓
python3 trader.py llm-open --symbol=XXX       # LLM分析开仓
# =============================================
# 交易执行（实盘无需 SIMULATE=true）
# =============================================
python3 trader.py open-long 10 --symbol=XXX --leverage=10      # 开多
python3 trader.py open-short 10 --symbol=XXX --leverage=10      # 开空
python3 trader.py cancel-conditionals --symbol=XXX                   # 取消指定币种全部追踪委托
python3 trader.py cancel-conditionals --symbol=XXX --side=SELL        # 只取消某一方向
python3 trader.py close-long --symbol=XXX                                         # 平多
python3 trader.py close-long --symbol=XXX --percent=50                        # 平多(减仓50%)
python3 trader.py close-short --symbol=XXX                                        # 平空
python3 trader.py close-short --symbol=XXX --percent=50                       # 平空(减仓50%)
```

## ⚠️ open-long / open-short 命令格式（关键！）

```bash
# ✅ 正确格式：保证金是位置参数，放在命令名后面

                            ^ 位置参数，不带 --


# ❌ 错误格式：--margin 会报 "unrecognized arguments"
```

| `--symbol` | 是 | 选项 | 币种，如 `NILUSDT` |
| `--leverage` | 否 | 选项 | 杠杆倍数，默认 `10` |

## 🛡️ 自动止损止盈

> ⚠️ **开仓成功后自动设置 2% 止损（不含杠杆）作为兜底保护，LLM 判断盈利后接管，重新设置止盈/新止损**

> ⚠️ **PM 账户限制**：API 无法查询/取消条件单，取消失败不影响设置新单（会自动覆盖）

| 操作 | 止损 | 后续止盈 |
|------|------|---------|
| **做多（open-long）** | 跌破入场价 2% 时触发，市价平仓 | LLM 分析盈利后重新设置 |
| **做空（open-short）** | 涨过入场价 2% 时触发，市价平仓 | LLM 分析盈利后重新设置 |

### 流程说明

**1. 开仓即设2% 止损**
- 这是地板保护，防止开盘就爆
- 止损触发后市价平仓

**2. LLM 分析后接管**
- 当持仓出现盈利，LLM 调用 `llm-hold` 分析盘口
- 根据 K线形态、成交量、关键阻力位重新判断止损位
- 大模型只需给程序一个参数+价格，程序自动取消旧单并下新单

**3. replace-order 命令（自动取消旧单后下新单）**

> ⚠️ PM 账户限制：取消可能返回 "Unknown order"（Binance 端未同步），但不影响设置新单
### LLM 接管后的止盈/止损操作（直接设置）

> ⚠️ 一键命令：自动取消旧单 + 设置新单




```bash

# 设置止损（程序自动获取持仓，全仓）（跌破触发）
python3 trader.py replace-order --symbol=NILUSDT --side=SELL --price=0.070 --type=STOP_MARKET
```

**参数说明：**

| 参数 | 必须 | 说明 |
|------|------|------|
| `--symbol` | ✅ | 币种，如 `NILUSDT` |
| `--side` | ✅ | `SELL`=平多仓 / `BUY`=平空仓 |
| `--price` | ✅ | 新触发价格（LLM根据盘口判断） |
| `--type` | 否 | `STOP_MARKET`(止损) / `TAKE_PROFIT_MARKET`(止盈)，默认 `STOP_MARKET` |
| `--algo-id` | 否 | 旧单ID（省略则程序自动从 `.conditional_orders.json` 查找） |

**追踪机制：**
- 活跃 algo_id 保存到 `.conditional_orders.json`（按 symbol + side 索引）
- 每次 replace 自动查找旧单 → 尝试取消 → 下新单 → 更新追踪文件
- 取消失败（可能已触发）不影响下新单

**4. cancel-conditionals 命令（按币种取消追踪的委托）**

```bash
# 手动平仓时同时取消全部追踪委托（SELL + BUY）
python3 trader.py cancel-conditionals --symbol=NILUSDT

# 只取消 NILUSDT 的 SELL 委托（平多），保留 BUY
python3 trader.py cancel-conditionals --symbol=NILUSDT --side=SELL
```

**使用场景：**
- 需要手动干预时，先取消程序追踪的旧单
- 换方向交易前，先清掉旧的止损
- 程序重启后，追踪文件仍保留历史 algo_id（重复取消无害）

##### 总得分计算 （必须算，不能凭感觉）

空头总得分（1-100分） = K线权重40% + 成交量权重40% + 多维度思考20%
多头总得分（1-100分） = K线权重40% + 成交量权重40% + 多维度思考20%
##### K线权重（各时间周期）

每个周期独立评分（0-100分），得分 × 权重 = 该周期最终贡献分。

**评分维度（各周期通用，满分100）：**

| 评分项 | 满分 | 打分标准 |
|--------|------|---------|
| 趋势结构 | 40分 | 均线（MA7/MA25/MA99）排列方向 + 趋势线方向 + 与更高周期方向一致性。趋势与交易方向一致+40，相反-30，震荡+15 |
| 关键K线形态 | 30分 | 出现与交易方向一致的信号形态（做多看涨/做空看跌）+20~30，无明确信号+10，出现反向信号-20 |
| 关键位置博弈 | 30分 | 当前价所处位置：即将向交易方向突破关键位且K线有力度+25~30，远离关键位置+10，关键位博弈失败/假突破-20 |

**各周期权重表：**

| 时间周期 | 权重 | 侧重说明 |
|---------|------|---------|
| 24h | 10% | 定大方向（是否与交易方向一致） |
| 4h | 20% | 看中期趋势延续性 + 主力建仓/出货痕迹 |
| 1h | 40% | 判断入场时机 + 量价配合是否到位 |
| 5m | 30% | 抓精确入场点 + 成交量爆发信号 |

**打分示例（做多方向）：**
- 4h周期：趋势多头排列（+40），出现吞没阳线突破均线（+25），距前高阻力仅3%（+25）→ 90分 × 20%权重 = 18分
- 若4h趋势空头排列（-30→趋势结构得40-30=10），出现射击之星（-20→K线形态得30-20=10），关键位博弈失败（-20→位置博弈得30-20=10）→ 10+10+10 = 30分 × 20% = 6分

**打分示例（做空方向）：**
- 4h周期：趋势空头排列（+40），出现吞没阴线跌破均线（+25），距前低支撑仅2%（+25）→ 90分 × 20% = 18分
- 若4h趋势多头排列（-30），出现锤子线（-20），关键位假跌破（-20）→ 30分 × 20% = 6分

**扣分项（任何周期出现即扣分，不设上限）：**
- 插针/假突破影线吞噬：-20分
- 连续3根以上同向衰竭K线（上影线递增阳线/下影线递增阴线）：-15分
- 放量滞涨（做多时阳线收不高）/放量滞跌（做空时阴线跌不深）：-20分

##### 成交量权重（各时间周期）

每个周期独立评分（0-100分），得分 × 权重 = 该周期最终贡献分。

**评分维度（各周期通用，满分100）：**

| 评分项 | 满分 | 打分标准 |

##### 多空倾向
统计做多和做空交易的胜率，倾向于胜率高的方向。

#### 策略模式
根据总资金多少，分为两种模式：

1. 小资金模式：激进，胆大心细，快进快出，见好就收，持仓数最多2
2. 大资金模式：保守，严格控制仓位，防止前功尽弃，长期趋势，持仓数最多5

#### 执行决策

- **开仓**：
  1. 根据总资产、胜率、仓位比例、策略模式计算出合理的仓位比例（20-80%）、杠杆倍数 `--leverage`（1-10），然后执行开仓
  2. 检查确认是否开仓成功 (必须回报有没有成功，如果没有就重试)
- **观望**：
  1. 如果没有可交易的币种（无信号），则扩大范围：`--top 10-50`，`--klines 10-50` ，`--min-vol 1.0-2.0`

#### 第三步：止损检查（严禁使用杠杆后盈亏）

**阶梯止损（Stepped Trailing TP）核心逻辑**
> - 激进策略=0.99做多，1.01做空
> - 保守策略=0.97做多，1.03做空
> ⚠️ **核心公式**（浮盈不含杠杆）：
> - 激活条件：浮盈 >= 2%
> - TP_trigger_price = 当前价 × 策略模式（做多）
> - TP_trigger_price = 当前价 × 策略模式（做空）
> - **触发条件**：当前价 <= TP_trigger（做多）| 当前价 >= TP_trigger（做空）
> - 触发后以市价平仓，锁住利润

**含义**：浮盈每增加 1%，TP_trigger 上移 1%。价格继续上涨时 TP 不触发；价格一旦回落，TP 立即锁住已累计的利润。利润越高，锁住的越多。
> 💡 每 1% 浮盈递增 1% 锁利。浮盈 2% 时锁 1%，3% 时锁 2%，4% 时锁 3%，依此类推。

```bash

# 设置止损（程序自动获取持仓，全仓）（跌破触发）
python3 trader.py replace-order --symbol=NILUSDT --side=SELL --price=0.070 --type=STOP_MARKET
```

**止损检查（按顺序执行，每步必须给出结论）**

| 检查项 | 触发条件（不含杠杆）| 执行动作 |
|--------|-------------------|---------|
| 初始止损 | 开仓即设（2% 地板止损） | 防止开盘爆仓 |
| 保本止损 | 浮盈 >= +1% 且止损仍在入场价以下 | 执行 设置止损 命令 将止损上移至当前价 × 0.99 |
| 阶梯止损更新 | 浮盈 >= 2%（首次激活） | 激活阶梯 TP，持续跟踪 |
| 阶梯止损更新 | 浮盈突破新阶梯（如从 3% → 4%）| 执行 设置止损 命令 自动更新 TP_trigger 到新阶梯 |

| 做空止损订单检查 | 计算止损价格小于当前价格 | 重新计算设置止损｜
