## 📋 交易命令

```bash
cd ~/.openclaw/workspace/trading_program

# =============================================
# 扫描候选币种（多线程并发，推荐配置）
# =============================================
python3 trader.py scan-all --top 10 --min-vol 3.0 --klines 10

# =============================================
# 账户 & 市场数据
# =============================================
python3 trader.py status                        # 账户状态
python3 trader.py market --symbol=BTCUSDT     # 市场数据
# =============================================
# LLM 分析
# =============================================
python3 trader.py llm-hold --symbol=XXX      # LLM分析持仓
python3 trader.py llm-open --symbol=XXX       # LLM分析开仓
# =============================================
# 交易执行（实盘无需 SIMULATE=true）
# =============================================
python3 trader.py open-long 10 --symbol=XXX --leverage=10      # 开多
python3 trader.py open-short 10 --symbol=XXX --leverage=10      # 开空
python3 trader.py cancel-conditionals --symbol=XXX                   # 取消指定币种全部追踪委托
python3 trader.py cancel-conditionals --symbol=XXX --side=SELL        # 只取消某一方向
python3 trader.py close-long --symbol=XXX                                         # 平多
python3 trader.py close-long --symbol=XXX --percent=50                        # 平多(减仓50%)
python3 trader.py close-short --symbol=XXX                                        # 平空
python3 trader.py close-short --symbol=XXX --percent=50                       # 平空(减仓50%)
```

## ⚠️ open-long / open-short 命令格式（关键！）

```bash
# ✅ 正确格式：保证金是位置参数，放在命令名后面
python3 trader.py open-long 10 --symbol=XXX --leverage=10
                            ^ 位置参数，不带 --


# ❌ 错误格式：--margin 会报 "unrecognized arguments"
python3 trader.py open-long --margin 10 --symbol=XXX
```


| 参数 | 必须 | 类型 | 说明 |
|------|------|------|------|
| `margin` | ✅ | 位置参数 | **保证金金额（USDT）**，不带 `--` |
| `--symbol` | 是 | 选项 | 币种，如 `NILUSDT` |
| `--leverage` | 否 | 选项 | 杠杆倍数，默认 `10` |

### 步骤1：获取数据

> 纯技术操作，不分析，只收集。**先获取持仓状态，根据持仓数决定是否扫描候选币种。**

#### 1.1 账户数据 + 判断需要什么数据
```bash
python3 trader.py status
```
- 实盘/虚拟账户、**总资产（记录备用，用于仓位换算）**、持仓币种、盈亏、持仓数

**判断流程**：
```
持仓数 == 0  → 空仓：执行1.2扫描候选币种 + 1.4获取全量memory → 对候选币种逐一分析（步骤3）
持仓数 < 上限 → 有仓但未满：执行1.2扫描（同时管理现有持仓）+ 1.4获取全量memory
持仓数 >= 上限 → 满仓：跳过扫描，只管理现有持仓 + 1.4获取全量memory
```

#### 1.2 扫描候选币种（空仓/未满仓时执行）

```bash
python3 trader.py scan-all --top 10 --min-vol 3.0 --klines 10
```
- **scan-all 的输出已包含每个候选币种的完整K线数据**（各周期均按 lg.md 数据规范的根数输出）
- 记录扫描结果中的候选币种列表，后续分析时直接复用 scan-all 打印的数据
- **如果候选币种全部被判断为"不做"，下一次扫描时要扩大范围：**
- 扩大 `--top`（10→20→30→50），看更多币种
- 降低 `--min-vol`（3.0→2.0→1.0），放宽成交量门槛
- 每轮空仓就加大一档，直到找到可交易的币种为止

> **满仓时跳过此步**，直接进入仓位管理流程。

#### 1.3 市场数据（补数据用）

```bash
python3 trader.py market --symbol=XXX
```
- **仅当** scan-all 的输出数据不够用时才调用（如 scan-all 数据过期了、或你想看更详细的周期）
- **扫描候选场景**：直接复用 scan-all 的数据，不需要每个币再调一次 market
- **持仓巡检场景**：获取持仓币种的实时K线数据供步骤3分析

#### 1.4 全量历史记忆（预判验证 + 交易记录）

```bash
cat ~/.openclaw/workspace/trading_program/memory/*.md
```

> **核心原则**：记忆记录的是你过去的预判和交易结果，用来帮你改进**预判能力**，不是限制你入场的。每轮必须从候选币种中选出一个进行重点监控（可交易可观察），保持对市场的跟踪不间断。**你的预判准确率是在实战中练出来的，连续观察也是在积累判断力。最终判断以当前K线数据为准，记忆只是参考。**

**读取记忆的目的**（不是找不做的理由，是看预判能力怎么提高）：
1. **看预判准确率** — 对每笔历史的预判验证，看自己的预判能力在什么水平。方向预判准确率越高说明判断力在进步，方向常错说明要调整分析角度
2. **看预判偏差模式** — 是方向判断问题（趋势识别错），还是时机问题（方向对但入场点不对），还是持仓问题（方向对但没拿住）？找到自己的弱项，这轮重点改进
3. **按币种看预判记录** — 当前币种过去预判了什么、实际怎么走的、预判准确度如何？这次分析时避开同样的偏差
4. **看交易定性** — 上次交易是"依据成立的正常交易"还是"预判有偏差的试错"？正常交易亏损→市场问题无需自责；试错亏损→预判方法需要改进
5. **多空胜率统计** — 只用于方向参考，**无论胜率高低都要基于当前数据独立判断**
6. **上次重点事项** — 上次说下次要在哪个环节做更好？这次做到了吗？

**机会驱动思维**：
- 核心问题不是"我错在哪里"，而是"我这次怎么预判才能更准"
- 历史记录帮我看到自己的预判偏差模式→这轮入场时针对性地避免这个偏差
- 不要因为没有十足把握就不做：市场没有100%准确的预判，小额试错是练预判的唯一方式
- 如果所有币种都判断"不做"，反问自己：是市场真没机会，还是我不敢验证自己的预判？

### 步骤4：执行

#### 4.1 开仓

**执行前确认**：
- 步骤3输出方向=做多或做空 → 执行
- 步骤3输出方向=不做 → 跳过
- 读取 `blacklist.json`（禁止交易的下架币种）

**仓位和杠杆**：
- 完全由 LLM 在步骤3的分析中自行决定仓位比例（0-100%）和杠杆倍数
- 程序按 LLM 指定的仓位和杠杆执行，**换算公式**：
  ```
  开仓保证金(USDT) = 总资产 × 仓位%
  例：总资产1000U, 仓位50%, 杠杆10x
     → 开仓保证金 = 1000 × 50% = 500 USDT
     → 命令：trader.py open-long 500 --symbol=XXX --leverage=10
  ```
  > 注意：**仓位%是基于总资产的保证金比例，不是名义仓位**。杠杆独立传给 `--leverage` 参数，不要乘以杠杆算保证金。
- **0仓位规则**：LLM 说50%就开50%，说80%就开80%。没有任何强制仓位公式，但 LLM 必须自己给出明确的仓位百分比和杠杆倍数。

**命令**（注意 `X` 替换为公式算出的保证金金额）：
```bash
python3 trader.py open-long X --symbol=XXX --leverage=X      # 开多
python3 trader.py open-short X --symbol=XXX --leverage=X      # 开空
```
- 确认开仓成功，否则重试
- 开仓后自动设置2%地板止损

---

#### 4.2 持仓管理

执行以下检查
> 持仓期间每次巡检按顺序检查：

**检查1：开仓理由还在吗？（按lg.md的持仓与止损逻辑）**
- 回顾开仓时的分析逻辑，新数据还支持吗？

**检查2：硬止损** — 逻辑与事实背离 → 立即全平

**检查3：阶梯止损**
> 核心规则：**每赚1%就往盈利方向移1%止损，保本后不回撤。** 开仓即设2%硬止损→浮盈≥1%移保本→之后每多赚1%止损就向盈利方向移1%，直到触发。
> 浮盈后调整止损用简化命令，程序自动算触发价：
> 多单（浮盈2%时锁1%）：`python3 trader.py replace-order --symbol=XXX --side=SELL`
> 空单（浮盈3%时锁2%）：`python3 trader.py replace-order --symbol=XXX --side=BUY`
> 自动计算规则：SELL=市价×0.99（下跌1%触发），BUY=市价×1.01（上涨1%触发）

**止损检查（按顺序执行，每步必须给出结论）**

| 检查项 | 触发条件（不含杠杆）| 执行动作 |
|--------|-------------------|---------|
| 初始止损 | 开仓即设（2% 地板止损） | 程序自动设置，无需执行 |
| 保本止损 | 浮盈 >= +1% 且止损仍在入场价以下 | 调用 设置止损 命令 将止损上移至入场价 |
| 阶梯止盈更新 | 浮盈 >= 1%（首次激活） | 激活阶梯，持续跟踪 |
| 阶梯止盈更新 | 浮盈突破新阶梯（如从 2% → 4%）| 调用 设置止损 命令 自动更新到新阶梯 |
| 阶梯止盈触发 | 当前价 <= 止损价 | 市价全平，汇报"触发阶梯止盈" |
| 止损价格检查 | 做多：止损价 > 当前价格 做空 止损价 < 当前价格| 重新设置 |

## 🏦 统一账户（PM_2）交易用法

- `trader.py` 自动检测账户类型并路由，无需手动切换
- 统一账户使用 `papi/v1/um/order` 接口（单向持仓）
- PM账户支持止损止盈条件单（papi/v1/um/algo/order）
- ⚠️ PM账户**无法通过API查询**活跃条件单（接口返回404），`get_conditional_orders()` 不可用
- `replace-order` 和 `cancel-conditionals` 通过 `.conditional_orders.json` 文件追踪 algo_id，自动处理旧单取消
