# lg.md — 市场语言词典

> **本文档只描述"市场长什么样"，不告诉 LLM "应该怎么做"。**
> 决策权在 LLM。SKILL.md 提供工具命令。本文是给 LLM 用的"市场词典"——方便 LLM 读懂 K 线语言。
>
> **没有任何"开仓/止损/平仓"硬规则**——LLM 看懂市场后自己决定。

---

## 0. 核心原则（7 条不变）

1. **价格 + 成交量是真实博弈**：OHLCV 5 维是市场唯一客观数据。
2. **多尺度时间**：同一币种不同尺度（1d/4h/1h/15m/5m）独立读，跨尺度一致才算确认。
3. **多视角集成**：趋势派 / 量价派 / 结构派，三个视角独立分析。
4. **反向视角**：每个决策先列 3 个反向理由，列举 ≥ 2 个强反驳 → 置信度降一档。
5. **IC 闭环**：每次预判后记录"预测 vs 实际"，方向准确率作为置信度参考。
6. **风险意识**：仓位大小自己决定，没有"必须 5% 止损"这种硬规则。
7. **持续学习**：亏本是策略问题，**改 lg.md 的视角描述**，不靠程序硬编码。

## 1. 多尺度时间窗口

市场在不同时间尺度上呈现不同结构。LLM 可以自由选择看哪些尺度。

| 尺度 | 数据频率 | 回顾窗口 | 适合观察什么 |
|------|---------|---------|------|
| 1d | 1d K 线 | 40 根 | 5 周大方向（趋势 / 反弹 / 顶部 / 底部）|
| 4h | 4h K 线 | 90 根 | 半月中期结构（强势 / 弱势 / 横盘）|
| 1h | 1h K 线 | 80 根 | 3 天节奏（脉冲 / 回调 / 持续）|
| 15m | 15m K 线 | 160 根 | 40 小时近期（入场时机）|
| 5m | **5m K 线** | 288 根 | 1 天微观（主力动作 / 量能）|

> **5m 是基准频率**：[Kronos 论文 arXiv:2508.02739](https://arxiv.org/abs/2508.02739) 训练数据是 5m K 线。**只用 1h/4h/1d K 线是"脉冲平滑"**——大量微观信号被滤掉，出现"全场横盘"假象。
> 
> 一个交易日 5m 有 288 根，1h 只有 24 根——**信息量 12 倍差距**。

**怎么用**：LLM 自由组合。常见组合：
- **看大方向**：1d + 4h
- **找入场时机**：15m + 5m
- **全视角**：1d / 4h / 1h / 15m / 5m 五个都看

**跨尺度一致才算确认**（1d 看多 + 4h 看空 = 矛盾 → 观望）——**这是视角描述，不是规则**。

---

## 2. 多视角集成

每个币种**强制从三个视角独立看**（不强制都得出同样结论，但每个视角都要分析）：

### 2.1 三个强制视角

| 视角 | 关注点 | 关键问题 |
|------|--------|---------|
| **趋势派** | 价格趋势 + 动量 | 高低点抬高/降低？趋势线方向？动量增强？|
| **量价派** | 成交量 + 资金流 | 放量/缩量与价格配合？量价背离？|
| **结构派** | 支撑/压力 + 形态 | 关键位置守住/突破？形态完整？|

### 2.2 视角结论怎么用

- 三视角**都看多** → 强多头信号
- 三视角**都看空** → 强空头信号
- 三视角**矛盾** → 观望
- 两视角一致 + 第三个中立 → 中等信号
- **没有"必须三视角都一致才开仓"这种硬规则**——LLM 自主权衡

### 2.3 反向视角（强制）

每个决策前**必须**先列反向理由：
- 决定做多 → 假设空头，列举 3 个做多会失败的理由
- 决定做空 → 假设多头，列举 3 个做空会失败的理由
- 列举 ≥ 2 个强反驳 → **置信度主动降一档**（这是软建议）

## 3. 量价状态识别

成交量是市场资金流的最真实记录。**没有"放量就是好/坏"的硬规则**——LLM 自由解读。

### 3.1 量能状态

| 状态 | 描述 | 关注点（LLM 自己看）|
|------|------|------|
| 放量 | 当前 5m vol 显著高于近期均量 | 是否与价格方向一致？|
| 缩量 | 当前 5m vol 显著低于近期均量 | 是否在关键位置？|
| 量价背离 | 价格新高但 vol 没新高 / 反之 | 主力意图？|
| 持续缩量 | 连续多根 5m vol 递减 | 趋势衰竭？|

**注**：扫描时已经给出"5m 摘要 vol=原始数字"（12 个数字逗号分隔）。**LLM 自己看数字判断放量/缩量**——程序不替 LLM 算"量比"或"atrpct"。

### 3.2 量能配合

- **放量上涨** = 多头主动进攻（强势）
- **放量下跌** = 空头主动进攻（弱势）
- **缩量上涨** = 多头乏力（可能是拉高出货）
- **缩量下跌** = 空头乏力（接近底部）
- **没有"必须看到某种量能才开仓"的规则**——LLM 自己判断

## 4. 5 维度集成（数据维度）

LLM 拿到的每根 K 线有 5 个数据维度：

| 维度 | 论文定义 | 我们的处理 |
|------|------|------|
| **Open / High / Low / Close** | 论文核心 4 列 | K 线原始数字（必给）|
| **Volume** | 论文说"optional" | 原始数字（**不离散化**，不替 LLM 算）|

**LLM 自己解读 5 维度的关系**——程序不替 LLM 做"形态识别"或"token 化"。

### 4.1 形态词典（LLM 自由用）

| 形态 | 描述 | 信号方向（LLM 自判）|
|------|------|------|
| 十字星 | O≈C，影线长 | 犹豫 / 转折 |
| 大阳线 | C-O > 3% | 多头强势 |
| 大阴线 | C-O < -3% | 空头强势 |
| 长上影 | U 影线 >> L 影线 | 上方抛压 |
| 长下影 | L 影线 >> U 影线 | 下方支撑 |
| 吞没 | 后 K 线实体包前 K 线 | 趋势反转 |
| 锤子线 | 小实体 + 长下影 | 底部信号 |

**注**：这些都是"LLM 看到后知道在说什么"的词典。**没有"看到锤子就开仓"这种规则**。

## 5. 置信度与 IC 评估

LLM 的"信心"靠**自己的多视角分析 + 反向视角**决定。**没有"S > 0.7 才开仓"这种硬规则**。

### 5.1 IC 是什么

IC (Information Coefficient) = 方向准确率（最近 N 笔的胜率）。是**事后回看**的指标，不是事前信号。

LLM 可以参考 IC 状态调减**仓位大小**（不是"拒交易"）：

- IC 高（≥50%）→ 仓位正常
- IC 中（40-50%）→ 仓位减半（lg.md 不禁止，程序自动 × 0.5）
- IC 低（30-40%）→ 仓位 × 0.25
- IC 极低（<30%）→ 仓位 × 0.1（试探仓）

**注意**：IC 状态是**仓位参考**，**不是开仓门槛**。IC 0% 也能开仓（"试错阶段"原则）。

### 5.2 动态权重（仅作参考）

`ic-weights` 命令输出"动态权重表"（1d/4h/1h/15m 权重），是给 LLM 的**分析侧重参考**——比如"近期 1h 比 1d 准，决策时 1h 多看几根"。

**没有"必须按权重表开仓"的规则**——LLM 自己决定侧重。

## 7. 移动止损阶梯（主流程的一部分）

**这不是硬规则，是 LLM 必走的"持仓巡检"动作**——和"开仓要设止损"一样，开仓后每次 cron 都要看一遍持仓有没有到阶梯档。

### 7.0 核心原则：止损只上不下（v6 2026-06-09）

**用户原话**："lg 止损 要只上不下，不然还怎么叫止损，不要跟着浮动"

**什么意思**：
- 止损触发价只能往"保护更多利润"方向走，**不能回扣**
- **LONG**：止损价 = `max(历史最高锁价, 本轮 current × 0.98)`
- **SHORT**：止损价 = `min(历史最低锁价, 本轮 current × 1.02)`
- 一旦"锁住"某价，后续即使浮盈回落（从 5% 跌回 3%），**也不动止损价**

**为什么不是"跟着 current 浮动"**：
- 跟着浮动 = 每轮都重算 `current × 0.98` = 跌下去时止损也下调 = 前面锁住的利润白锁了
- 只上不下 = 止损价只动一次（往上），下后再跌也不会推低止损 = 锁住的利润"被锁死"

**示例**（LONG 入场 100）：
- 浮盈 2%（102）→ 保本：止损 = 100
- 浮盈 5%（105）→ target = 105 × 0.98 = 102.90，新高，**锁档 102.90**
- 浮盈 3%（103）→ target = 103 × 0.98 = 100.94，**低于 102.90，不动，仍是 102.90**（锁住不返档）
- 浮盈 6%（106）→ target = 106 × 0.98 = 103.88，新高，**锁档 103.88**
- 浮盈 4%（104）→ target = 104 × 0.98 = 101.92，**低于 103.88，不动，仍是 103.88**

**持久化**：每个 `symbol+side` 的"最高锁价"持久化到 `.ladder_peak.json`，跨 cron 进程依然有效。`do_close_long/short` 成功时由程序主动清理。

### 7.1 阶梯档位（主流程：2% 保本, 3% 锁 1, 4% 锁 2, 5% 锁 3）

| 浮盈 | 锁档 | 心理锁 | LONG target | SHORT target | 含义 |
|------|------|--------|----------------|------------------|------|
| < 2% | — | — | (沿用开仓 3%) | (沿用开仓 3%) | 未到档，不动 |
| ≥ 2% | 保本 | 锁 0% | `entry` | `entry` | 不亏钱 |
| ≥ 3% | 锁 1% | +1% | `current × 0.98` | `current × 1.02` | 锁 1% 利润 |
| ≥ 4% | 锁 2% | +2% | `current × 0.98` | `current × 1.02` | 锁 2% 利润 |
| ≥ 5% | 锁 3% | +3% | `current × 0.98` | `current × 1.02` | 锁 3% 利润 |
| ≥ 6% | 锁 4% | +4% | `current × 0.98` | `current × 1.02` | 锁 4% 利润 |
| ≥ 7% | 锁 5% | +5% | `current × 0.98` | `current × 1.02` | 锁 5% 利润 |
| ≥ 8%+ | 锁 N% | +N% | `current × 0.98` | `current × 1.02` | 8 锁 6, 9 锁 7, 10 锁 8... |

**target 公式（本轮预计，仅供参考）**：
- 浮盈 ≥ 3%：`target = current × 0.98` (LONG) / `current × 1.02` (SHORT)
- 保本档（2%）：`target = entry`
- < 2%：不动

**实际止损价 = max(历史锁价, target)**（LONG 场景） / **min(历史锁价, target)**（SHORT 场景）

### 7.2 主流程：开仓后必执行

```
1. 开仓成功 → 程序自动挂 3% 紧止损 (entry × 0.97 / entry × 1.03), 写入 .conditional_orders.json
2. 每次 cron status 后 → 调 check-ladder
3. check-ladder 扫所有持仓:
   a. 读 .ladder_peak.json 拿历史最高锁价
   b. 算本轮 target = current × 0.98 (LONG ≥ 3%) / entry (保本档) / 不动 (< 2%)
   c. 对比: target 优于历史锁价 → 建议 replace-order (并更新 peak)
              target 不优于历史锁价 → 不动 (锁住不返档)
4. 建议出来的才调 replace-order --trigger=<建议价>
5. 阶梯只上不下，浮盈跌回去不动
```

### 7.3 命令对照

| 动作 | 命令 | 说明 |
|------|------|------|
| 看阶梯档 | `python3 trader.py check-ladder` | 输出所有持仓的浮盈 + 阶梯档 + 建议 trigger + peak 锁档状态 |
| 调止损 | `python3 trader.py replace-order --symbol=X --side=SELL/BUY --trigger=Y` | 不传 --trigger 默认 current × 0.98 / × 1.02；传 --trigger 覆盖 |

### 7.4 为什么不写进 SKILL.md 当硬规则

**lg.md 写"主流程的一部分"（必走动作）** vs **SKILL.md 写"3% 紧止损"（程序硬约束）**——分得清：
- lg.md 告诉 LLM "开仓后要看阶梯"（**必走**）
- lg.md 不告诉 LLM "必须 3% 起步"（那是 SKILL.md 的事，程序自己挂）
- lg.md 不告诉 LLM "阶梯到第几档该平仓"（那是 LLM 自主决定）

**一句话**：阶梯是**持仓后的保护动作**，不是开仓的**入场条件**。LLM 决定"开不开仓"，开了之后程序/LLM 必走巡检。

## 14. 持仓分析（"几时平仓"的预测视角）

**止损是保底，不是平仓信号**。lg.md §7 阶梯只负责"保本/锁利"，但**主动平仓** 要靠预测。

### 14.1 为什么需要"持仓分析"

上次事故（2026-06-09 INUSDT -10%）：止损单被 status 误删 + LLM 只盯着阶梯档位，**没主动预判 INUSDT 从 +2% 跌到 -10% 这个趋势反转**。

机械触发（阶梯）会清金字塔顶，但**趋势反转 / 动能衰退 / 量能价量背离 这些"主动平仓信号"得靠 LLM 预测**。

### 14.2 每个持仓必问 4 个预测问题

拿到一个持仓后，**必须**逐一回答以下 4 个问题（用 5 尺度 + 量价多视角集成）：

| # | 预测问题 | 怎么分析 |
|---|---------|----------|
| 1 | **当前趋势是否在走完**？ | 1d/4h 高点是否不再抬高？1h/15m 是否跌破近期支撑？动能 (C-O) 实体是否在缩小？|
| 2 | **动能是否衰退**？ | 阳线 C-O 缩窄 / 阴线 C-O 变长 / 上下影变长 → 犹豫信号。最近几根 K 线是否有"长上影"或"长下影"？|
| 3 | **量价是否配合**？ | 价格新高但 vol 没新高 = 量价背离 → 趋势可能反转。价格低位盘整 + vol 持续缩量 = 多头乏力。|
| 4 | **反向视角的反驳是否强**？ | 列举 3 个"为什么现在不该平"的理由。如果 3 个都强 → 不平。如果 2 个以上被驳倒 → 平。|

### 14.3 主动平仓 vs 继续持有的判定框架

| 场景 | LLM 判断 | 动作 |
|------|---------|------|
| 4 个预测问题中 3 个以上"趋势在走完" | 主动平仓信号强 | 调 `close-long` / `close-short` |
| 浮盈 ≥ 3% + 阶梯锁档 + 动能衰退迹象 | 先锁档不卖，限价单主动出 | 调 `replace-order` |
| 4 个问题都"趋势还在" | 继续持有 | 不动 + 跟踪 |
| 浮亏 + 止损未穿 + 跨尺度信号未变 | 继续持有 | 不动 |
| 浮亏 + 止损已穿 | 止损 (程序已设) | 等触发 |
| 浮亏 + 止损未穿但 4 个预测问题 3+ 个"趋势反转" | 主动平仓（不等止损）| 调 `close-long` / `close-short` |

### 14.4 主动平仓的 3 个量价信号（重点）

1. **量价背离**：价格创新高 / 新低，但同期 vol 明显降低 → "虚突破" / "虚跌破"，主力可能反手
2. **动能耗尽**：连续 3+ 根 K 线 C-O 实体越来越小 + 影线变长 → 市场犹豫，多空均衡可能被打破
3. **关键位失守 / 突破**：4h/1d 的关键支撑 / 压力位被放量突破 或 失守 → 趋势变盘信号

### 14.5 命令对照

| 动作 | 命令 |
|------|------|
| 查持仓 + 阶梯档 | `python3 trader.py check-ladder` |
| 查 5 尺度 K 线 | `python3 trader.py market --symbol=X --interval 1d --interval 4h --interval 1h --interval 15m --interval 5m` |
| 锁档（被动保护）| `python3 trader.py replace-order --symbol=X --side=平仓方向 --trigger=Y` |
| 主动平仓 | `python3 trader.py close-long X` 或 `close-short X`（不传 qty = 全平）|

### 14.6 跟 §7 阶梯的关系

- **§7 阶梯** = 被动保护（"已赚到的别丢"）— LLM 必走，但它是机械的
- **§14 主动平仓分析** = 主动预判（"这个趋势能不能继续"）— LLM 必走
- **两者并行**：每次 cron 先 §7 巡检（被动），再 §14 主动分析（4 问 + 框架）

### 14.7 严禁只依赖阶梯

**只看阶梯 = 错过主动平仓机会**。例：BASUSDT 浮盈 +9% 触发阶梯锁 7%，但 LLM 看完 5 尺度后判断动能衰退、量价背离，应该主动 close-long 全平，而不是傻等阶梯把利润锁进。

**lg.md 不规定"什么时候必须平"**（避免硬规则反噬），但**§14.2 的 4 个预测问题** 是 LLM 必答的——不答 = 等同于没分析持仓。

### 14.8 analyze-position 命令（v6 2026-06-09 新增）

**前 3 问脚本化**：Q1 趋势 / Q2 动能 / Q3 量价 由程序自动算（拉 1d/4h/1h/15m 4 尺度 K 线 + 量化阈值判断），**Q4 反向支撑仍由 LLM 答**（需要上下文理解）。

```bash
python3 trader.py analyze-position --symbol XYZ
```

**输出**（含 4 问表格 + 判定）：

```
🔍 XYZ LONG 持仓分析 (lg.md §14)
入场: $0.0787  现价: $0.0722  浮盈: -8.18%  数量: 127.0

§14 4 问脚本化结果
Q1 趋势走完?    [走完]  1d 近 5 根高点不抬高 (现=0.07220, 近期 max=0.07870)
Q2 动能衰退?    [衰退]  4h 近 3 根 C-O 缩窄 (近3=0.12% < 前2=0.18%) + 影线/实体=0.85
Q3 量价背离?    [背离]  1d 后5 vol=0.47B < 前5 vol=1.21B (39%) 后5高点创新高=0.0722
Q4 反向支撑?    (LLM 填)  3 个"为什么不平"理由 — 需 LLM 列举

判定: 主动平仓  (weak_count=3/3, 浮盈=-8.18%)
```

**判定逻辑**：
- LONG:
  - `weak_count >= 3 且 浮盈 > 0` → **主动平仓**
  - `weak_count >= 2 且 浮盈 > 0` → **锁档**
  - `浮盈 < -3%` → **止损等待**（程序止损已设）
  - 其他 → **持有**
- SHORT: 同上

**LLM 必走**：调 `analyze-position`，把 4 问结果填进 §10 报表模板的"持仓 4 问分析"表。

### 14.9 §7 阶梯 + §14 4 问 的执行顺序

每次 cron 步骤 1 = §7 阶梯巡检 + §14 主动平仓分析**都要走**（不是二选一）：

```
步骤 1a: §7 阶梯 (机械化)         → check-ladder
步骤 1b: §14 主动平仓 (预测化)    → analyze-position × 每个持仓
步骤 2:  扫候选池 (scan-all)
步骤 3:  决策 (LLM 自主)
步骤 4:  写 memory (10.0 模板)
```
